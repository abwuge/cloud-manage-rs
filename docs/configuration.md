# Configuration Reference

This document describes the on-disk configuration files used by
cloud-manage-rs. Provider-specific notes live in the corresponding guide
under [providers/](providers/).

## Files

| Path | Purpose | Owner |
| --- | --- | --- |
| `./config/config` | Unified TOML: provider auth + instance + network. | Generated by the wizard / migration; safe to hand-edit. |
| `./config/oci_api_key.pem` (or any path) | OCI API private key referenced by `[oci].key_file`. | User-provided. |

### Legacy Migration

Earlier versions used two files: `./config/oci_config` (INI) and
`./config/instance_config.toml`. On startup, if `./config/config` does not
exist but both legacy files do, they are merged into the new format
automatically. The legacy files are left in place; you can delete them once
you have verified the merged output.

## `config`

```toml
[oci]
user        = "ocid1.user.oc1..xxx"
fingerprint = "aa:bb:cc:dd:ee:ff:00:11:22:33:44:55:66:77:88:99"
tenancy     = "ocid1.tenancy.oc1..xxx"
region      = "us-phoenix-1"
key_file    = "./config/oci_api_key.pem"
# passphrase = "..."  # optional

[cloudflare]
api_token = "cf_api_token_with_zone_dns_edit"
zone_name = "example.com"
record_name = "app" # optional default record

[oracle]
compartment_id      = "ocid1.compartment.oc1..xxx"
availability_domain = "Uocm:PHX-AD-1"
subnet_id           = "ocid1.subnet.oc1.phx.xxx"
image_id_amd        = "ocid1.image.oc1.phx.xxx"
image_id_arm        = "ocid1.image.oc1.phx.xxx"
ssh_public_key      = "ssh-ed25519 AAAA... user@host"

[instance]
instance_type       = "arm"        # "amd" | "arm"
display_name        = "my-instance"
arm_ocpus           = 2            # ARM only; 1..=4
arm_memory_gb       = 12           # ARM only; conventionally arm_ocpus * 6
boot_volume_size_gb = 47

[network]
assign_public_ip    = true
assign_ipv6         = true
private_ip          = "10.0.0.11"                  # optional
ipv6_address        = "2001:db8:1234:5678:11"      # optional
hostname_label      = "my-host"                    # optional

[snipe]
min_delay_secs      = 5.0
max_delay_secs      = 30.0
max_attempts        = 0      # 0 = unlimited
```

### `[oci]`

OCI authentication. The fields mirror the standard `~/.oci/config` INI file:
`user`, `fingerprint`, `tenancy`, `region`, `key_file`, optional `passphrase`.
See [Oracle guide](providers/oracle.md#auth-setup) for how to obtain them.

### `[cloudflare]`

Cloudflare DNS authentication. Only API token auth is supported; global API
keys and email/key auth are intentionally not represented in the config.
The UI accepts `record@zone`, such as `app@example.com`, and stores it as
`record_name = "app"` plus `zone_name = "example.com"`.

| Field | Type | Notes |
| --- | --- | --- |
| `api_token` | string | Cloudflare API token scoped to the target zone with **Zone:DNS Edit** permission. |
| `zone_name` | string | Cloudflare zone/domain name, for example `example.com`. The SDK resolves the zone ID online. |
| `record_name` | optional string | Default DNS record name used by the interactive DNS prompts. |

### `[oracle]`

Oracle-specific resource selection. Fields are documented in the
[Oracle guide](providers/oracle.md#configuration-block).

### `[instance]`

| Field | Type | Notes |
| --- | --- | --- |
| `instance_type` | `"amd"` \| `"arm"` | Selects the shape family. |
| `display_name` | string | Free-form name shown in the console. |
| `arm_ocpus` | int (1–4) | Used only when `instance_type = "arm"`. |
| `arm_memory_gb` | int | Used only when `instance_type = "arm"`. Conventionally `arm_ocpus * 6`. |
| `boot_volume_size_gb` | int | Boot volume size in GB. Provider minimums apply. |

### `[network]`

| Field | Type | Notes |
| --- | --- | --- |
| `assign_public_ip` | bool | Attach a public IPv4. |
| `assign_ipv6` | bool | Enable IPv6 on the primary VNIC. |
| `private_ip` | optional string | Private IPv4 in the subnet's CIDR. Omit to let the provider pick. |
| `ipv6_address` | optional string | Specific IPv6 in the subnet's IPv6 CIDR. Omit to auto-assign. |
| `hostname_label` | optional string | DNS hostname label for the VNIC. |

### `[snipe]`

Defaults for snipe mode (retry-until-success). The interactive snipe menu
uses these as the prompt defaults and persists any change back to the file;
the `cloud-manage snipe` CLI uses them when the corresponding flag is
omitted, and writes them back only when `--save` is passed.

| Field | Type | Notes |
| --- | --- | --- |
| `min_delay_secs` | float | Lower bound of the randomized delay between attempts. |
| `max_delay_secs` | float | Upper bound. If lower than `min_delay_secs` the two are swapped. |
| `max_attempts` | int | Stop after this many tries. `0` means unlimited. |

## Wizard Behaviour

- Defaults shown to the user are taken from the existing
  `./config/config` (if any) or the auth-section-derived fallbacks.
- An invalid value (placeholder, malformed OCID, or a fetch that fails) is
  treated as **not set**: the wizard falls back to the standard prompt
  rather than refusing to continue.
- Background prefetch:
  - Image details are fetched as soon as the wizard starts (when valid
    OCIDs are present in the config).
  - Availability domains and subnets are fetched using the candidate
    compartment (typically `[oci].tenancy`) before the user finishes
    typing the compartment OCID.
  - If the user picks a compartment different from the candidate, the
    prefetched results are discarded and the listings are refetched.

## Editing By Hand

`./config/config` is plain TOML and may be edited directly. The next run
will load it (or fall back to the wizard if parsing fails). Any field
considered invalid will simply be ignored, see above.
